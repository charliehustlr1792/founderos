Complexity level
Low–Medium
Marketplaces are technically moderate complexity — the core listing and search is straightforward, but the two-sided nature (artists + customers) adds auth and data modelling overhead. A focused developer can get a working v1 live in 6–8 weeks. The real complexity is in the UX — making discovery feel effortless is a product problem, not an engineering one.
Suggested approach
Next.js for frontend + Supabase for database, auth, and storage. Google Maps API for location. Cal.com embedded for booking. Vercel for hosting. No custom backend needed at v1 scale.

Common mistakes to avoid
⚠
Building for both sides at onceNail techs won't join without customers. Customers won't come without nail techs. Solve supply first — manually onboard 20 artists in one city before building the customer-facing product.
⚠
Building payments before proving the core loopStripe integration takes a week. Validate that users actually complete a booking first — then add payments once the behaviour is confirmed.
⚠
Launching nationally instead of in one cityLocal density is everything in a location-based marketplace. Pick one city, dominate it, then expand. A sparse national launch feels dead on arrival.
